# Programmiersprache Swift

Die Implementierung erfolgt primär mit Swift (Embedded).

## Entscheidung

✅ Swift 

## Optionen

✅ Swift - weil die Lernkurve extrem flach ist und die Sprachkonzepte trotzdem für eine OS Entwicklung (seit Embedded Swift) geeignet sind. Der *notwendige Assemblerteil wird auf das Minimum* beschränkt. Mit den modernen LLM können zudem die fehlenden Beispiele und Tutorials on-the-fly generiert werden. 

❌ Rust - weil die Lernkurve sehr hoch ist ohne dass ein elementarer Vorteil gegenüber Swift 6 gegeben ist. Vorteil von Rust ist, dass es gerade Hardwarenah eine sehr gute Community bei einem OS in Rust gibt und ein entsprechendes Projekt existiert. Ebenso hat Rust mit der Integration in Linux Kernel einen wesentlichen Schritt geschafft.

❌ C / C++ - weil es schon viele Projekte gibt und die Speichersicherheit dort fehlt. Vorteil von C / C++ ist die Unmengen von Tutorials und Beispiele die zur Verfügung stehen.

❌ Java - weil es sich wegbewegt hat von der Plattform für alles und die Komplexität inzwischen sich derer von C / C++ nähert ohne einen Mehrwert. Zudem fehlt es an einer guten (nahezu) freestanding Version, da es sich als Business Language Ersatz (für COBOL) positioniert hat. 

❌ Assembler - weil es an den Mehrwerten einer Hochsprache fehlt. 

❌ COBOL - weil es sich als Business Language positioniert hat.
